# Testkonzept

## Ziel
Die wichtigsten Nutzerflüsse der Terminbuchungs-Webapplikation werden funktional getestet, um sicherzustellen, dass Registrierung, Login, Terminbuchung, Stornierung und Praxisverwaltung korrekt funktionieren.

## Testarten
- Manuelle funktionale Tests
- Exploratives Testing
- Erste automatisierte Smoke-Tests mit Playwright

## Testumgebung
- VM: hosting-060.wecote.ls.eee.intern
- URL: http://hosting-060.wecote.ls.eee.intern/terminbuchung/
- Browser: Chrome / Chromium, Firefox, WebKit
- Datenbank: MariaDB / MySQL
- Backend: PHP auf Apache

## Fokus
- Patientenregistrierung
- Patientenlogin
- Terminbuchung
- Meine Buchungen
- Stornierung
- Praxislogin
- Admin-Dashboard
- Kalenderansicht
